# Security Policy

This repository contains a SysML v2 model and documentation. It is not deployed software, but security issues may still include malicious files, unsafe workflow changes, or misleading automation.

## Reporting A Vulnerability

Please do not disclose suspected security issues publicly before maintainers have had a chance to review them.

Report security issues through GitHub private vulnerability reporting when available, or contact the maintainers through the security contact listed by the Elan8 organization.

Include:

- affected file or workflow
- reproduction steps
- expected impact
- any suggested fix

Maintainers will acknowledge valid reports and coordinate remediation in the public repository when appropriate.
